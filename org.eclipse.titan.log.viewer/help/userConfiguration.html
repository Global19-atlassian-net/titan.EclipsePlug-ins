User Configuration


The TITAN Log Viewer user configuration can be configured on either project or workspace level. The settings on the workspace level are referred to as preferences and applies to all projects in the workspace, while settings on the project level are referred to as properties and applies to the project they are set on. Project settings overrides the workspace settings when used.

The workspace preferences are found in the Preferences in the Window menu while the project properties is found in the action Properties in the pop-up menu of the selected project.

The layout of the property/preference pages are similar on project and workspace level, except the option to set workspace or project level that only exists in the project properties. If the user chooses to use settings defined on workspace level in the properties, the project settings will be disabled.


All property/preference pages contain the same buttons, with the following functionality:

Button Description
Import Import opens a browser dialog for importing property/preference page settings.
Export Export will open a browser dialog for exporting property/preference page settings.
Apply The changes in the property/preference page is saved and the focus will remain in the window.
Restore Defaults Sets the values back to the default values.
OK Saves the settings in the window and closes the window.
Cancel Closes the window without saving the settings.


Export

The export functionality is available on each property/preference page, General Settings, Components Visual Order, Filtered Components, Filtered Functions, Filtered Signals and Filtered Silent Events .

 

The exported property/preference settings are stored in a XML file. The export functionality will export all settings that are visible in the property/preference page even if the have not been committed (e.g. stored).

The exported file is possible to transfer between different operating systems. A file exported from a project setting can be imported into the workspace setting and vice versa.

  1. Press the Export... button in the property/preference page.

    The export dialog is opened.

  2. Enter file name and output directory for the export file and press the Save button.

    The property/preference settings visible in the page is exported to a file with the extension xml with the selected name.

Import

The import functionality is available on each property/preference page, General Settings, Components Visual Order, Filtered Components, Filtered Functions, Filtered Signals and Filtered Silent Events .

 

This import functionality will import an existing XML file and visually display the data, the user must explicit save the content. The import can occur both on project and workspace level.

The file imported can have been created on a different operating system. A file exported from a project setting can be imported into the workspace setting and vice versa.

  1. Press the Import... button in the property/preference page.

    The import dialog is opened.

  2. Select the file to import and press the Open button.

    The property/preference settings visible in the page is exported to a file with the extension xml with the selected name.
